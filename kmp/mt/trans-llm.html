Классические системы машинного перевода (SMT, NMT) обучаются на параллельных корпусах — миллионах пар предложений «исходный текст → перевод». Архитектура encoder-decoder специально проектируется для задачи seq2seq-преобразования между языками.

Большие языковые модели (LLM) обучаются иначе: они предсказывают следующий токен на огромных массивах одноязычных текстов (с небольшой долей многоязычных данных). Специальной задачи «перевод» в претрейнинге нет.

Тем не менее, LLM демонстрируют впечатляющие способности к переводу — иногда превосходящие специализированные системы. Как это возможно?

Важно: Способность LLM к переводу — не результат прямого обучения, а эмерджентное свойство, возникающее из общих языковых способностей модели.

Эмерджентность в контексте LLM

Эмерджентность (emergence) — появление качественно новых свойств системы, которые не были явно запрограммированы и не сводятся к сумме компонентов.

Примеры эмерджентных способностей LLM:

  • Арифметика (модель не обучалась на примерах сложения)
  • Рассуждения по цепочке (chain-of-thought)
  • Следование сложным инструкциям
  • Перевод между языками
  • Программирование на языках с малым представительством в данных

Условия эмерджентности:

  • Достаточный масштаб модели (миллиарды параметров)
  • Разнообразие обучающих данных
  • Достаточная глубина архитектуры (слои внимания)
Пояснение: Эмерджентные способности часто проявляются скачкообразно при увеличении размера модели — феномен «фазового перехода».

Почему LLM умеют переводить? Гипотезы

Исследователи предлагают несколько объяснений (не взаимоисключающих):

1. Мультиязычное присутствие в данных

Хотя параллельных корпусов мало, в обучающих данных присутствуют:

  • Тексты на разных языках (Wikipedia, Common Crawl)
  • Code-switching и смешанные тексты
  • Переводческие примеры в веб-страницах
  • Многоязычные документы (EU, UN, инструкции)

2. Формирование «внутренней интерлингвы»

Гипотеза: в глубоких слоях модели формируется языконезависимое представление смысла — своего рода латентная интерлингва.

  • Исследования показывают сходство активаций для семантически эквивалентных предложений на разных языках
  • Модель «понимает» смысл, а затем «генерирует» его на целевом языке

3. Общие паттерны языкового устройства

Языки мира имеют общие структурные свойства (универсалии). LLM, обучаясь на множестве языков, захватывает эти универсалии и может применять их кросс-лингвистически.

4. In-context learning

LLM способны к обучению «на лету» по нескольким примерам в промпте. Перевод может быть частным случаем этой способности: модель «видит» формат задачи и выполняет её.

Примечание: Вопрос о внутренних представлениях LLM — активная область исследований (mechanistic interpretability). Однозначного ответа пока нет.

Отличия перевода LLM от классического MT

Преимущества LLM:

  • Гибкость: можно управлять стилем, регистром, целевой аудиторией через промпт
  • Контекст: большое контекстное окно позволяет учитывать дискурс
  • Мультизадачность: одна модель — перевод, редактура, объяснение, адаптация
  • Межсемиотичность: мультимодальные LLM работают с текстом, изображениями, аудио
  • Zero-shot: перевод для языковых пар без параллельных данных

Ограничения и риски LLM:

  • Галлюцинации: модель может «изобретать» контент, отсутствующий в оригинале
  • Непредсказуемость: результат зависит от формулировки промпта, температуры
  • Терминология: без специальной настройки хуже справляется с узкими доменами
  • Отсутствие гарантий: нельзя доказать корректность перевода
  • Ресурсоёмкость: требует значительных вычислительных мощностей

Эмерджентность перевода в LLM ставит под вопрос некоторые привычные представления:

1. Перевод — не изолированный навык

Возможно, способность к переводу — не отдельная компетенция, а проявление более глубокого «понимания» языка и смысла. LLM показывают, что достаточно мощная языковая модель «автоматически» приобретает переводческие способности.

2. Вопрос об интерлингве реабилитирован

Идея универсального языка-посредника (интерлингвы) считалась непрактичной. Но если LLM формируют внутренние языконезависимые представления — это фактически нейросетевая интерлингва.

3. Связь с уровнями понимания

  • Классические MT работают на уровнях 2–3 (грамматика, контекст предложения)
  • LLM потенциально оперируют на уровнях 6–8 (коммуникативная функция, гибкие инварианты)
  • Мультимодальные LLM — уровень 5 (межсемиотический перевод)

4. Размытие границ перевода

Для LLM нет принципиальной разницы между:

  • Межъязыковым переводом
  • Перефразированием
  • Стилистической адаптацией
  • Описанием изображения
  • Суммаризацией

Это подтверждает семиотический взгляд: всё это — варианты преобразования знаковых систем с сохранением выбранного инварианта.

Важно: LLM эмпирически подтверждают высокие уровни понимания: перевод — это не замена слов, а преобразование смысла между знаковыми системами.

LLM для перевода (рекомендации):

  • Формулируйте чёткие инструкции: целевая аудитория, стиль, домен
  • Предоставляйте контекст: предыдущий текст, глоссарий, примеры
  • Используйте few-shot примеры для нестандартных задач
  • Итерируйте: просите модель улучшить/исправить перевод

Как анализировать ошибки LLM:

  • Галлюцинации: контент, которого нет в оригинале (проблема уровня 8 — неверный инвариант)
  • Пропуски: модель «решила», что информация избыточна
  • Стилистические сдвиги: несоответствие регистра
  • Терминологическая непоследовательность

Критическое мышление:

  • LLM — не «понимающий» субъект, а буссубъектный вероятностный актор (агенс)
  • Эмерджентность ≠ сознание или (человеческая) интенциональность
  • Качество зависит от данных, на которых обучалась модель
  • Качество зависит от способов обучения и дообучения модели
  • Качество зависит от системного промпта и других ограничений модели
  • Для ответственных задач необходима человеческая верификация

Связь эмерджентности LLM с уровнями понимания

Уровень Способность LLM Источник способности
1–2 (лексика, грамматика) Высокая для ресурсных языков Статистические паттерны в данных
3 (прагматика) Хорошая, зависит от промпта Контекстное окно + разнообразие данных
4 (мультимодальный вербальный) Высокая (парафраз, упрощение) Общая языковая компетенция
5 (межсемиотический) Высокая в мультимодальных моделях Совместное обучение на текстах и изображениях
6 (коммуникативный) Средняя, требует явных инструкций RLHF, instruction tuning
7–8 (формальный, универсальный) Гибкая — инвариант задаётся промптом Эмерджентная способность к instruction following

Финиш. Ключевые тезисы

  • LLM не обучаются переводу напрямую — способность возникает эмерджентно
  • Это свидетельствует о глубокой связи между языковым моделированием и переводом
  • LLM потенциально работают на всех уровнях понимания, включая высшие
  • Гибкость достигается через промпт — инвариант можно задавать динамически
  • Эмерджентность не означает «понимания» в человеческом смысле
  • Критический анализ и верификация остаются задачей человека

Практический вывод:

Лингвист, понимающий природу LLM и их отличие от классических MT-систем, может:

  • Эффективнее использовать LLM для профессиональных задач
  • Лучше диагностировать и классифицировать ошибки
  • Участвовать в разработке и оценке новых систем
  • Вносить вклад в теоретическое осмысление природы перевода
Важно: Эмерджентность перевода в LLM — не только технологический феномен, но и теоретический вызов, требующий лингвистической рефлексии.
