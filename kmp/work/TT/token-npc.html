1Введение

В парадигме AI First важно познакомиться с проведением токенизации с помощью LLM как образовательных NPC (Non-Player Character), где моделируется пользователем LLM-персонаж с определённой ролью, характером и стратегией взаимодействия. Сам промптинг по заданию роли рекомендуется обсуждать с LLM.

Токенизация для лингвистов

Лингвисты привыкли работать с морфемами, лексемами, синтагмами — единицами, выделяемыми по лингвистическим критериям. Большие языковые модели (LLM) оперируют иными единицами — токенами, которые определяются вероятностно и не всегда совпадают с привычными лингвистическими границами. Понимание этого разрыва критически важно для:

  • грамотного использования LLM в преподавании и исследованиях;
  • осознания ограничений и «слепых зон» языковых моделей;
  • эффективного промптинга — составления запросов к LLM;
  • критической оценки результатов, которые выдаёт модель.

2Основы токенизации

Токенизация (tokenization) — процесс разбиения непрерывного текста на минимальные единицы (токены), которые служат входными данными для модели обработки естественного языка (NLP). Токен может соответствовать слову, части слова, отдельному символу или даже фрагменту байтовой последовательности.

Ключевое различие Лингвист спрашивает: «Какова структура и значение этой единицы?» Алгоритм токенизации спрашивает: «Как закодировать текст наименьшим числом единиц из фиксированного словаря?» Эти задачи пересекаются, но не совпадают.

Понимание разницы позволяет лингвисту точнее интерпретировать поведение LLM: модель не «знает» морфологию — она оперирует статистическими паттернами на уровне токенов.

3Виды токенизации

3.1. Пословная (word-level)

Текст разбивается по пробелам и знакам препинания. Каждое слово — отдельный токен.

  • Плюс: интуитивно понятна, совпадает с бытовым представлением о «словах».
  • Минус: огромный словарь (сотни тысяч словоформ для флективных языков), проблема «неизвестных слов» (OOV).

3.2. Посимвольная (character-level)

Каждый символ (буква, цифра, знак) — отдельный токен. Словарь минимален.

  • Плюс: нет проблемы OOV, маленький словарь.
  • Минус: последовательности очень длинные, модели сложнее «увидеть» семантику.

3.3. Подсловная (subword) — доминирующий подход

Компромисс: частотные слова остаются целыми, редкие разбиваются на подсловные единицы. Основные алгоритмы:

АлгоритмПринципИспользуется в
BPE (Byte Pair Encoding)Итеративное слияние наиболее частотных пар символовGPT-серия, LLaMA
WordPieceСлияние пар, максимизирующих правдоподобие языковой моделиBERT, DistilBERT
UnigramНачинает с большого словаря, итеративно удаляет наименее полезные единицыT5, ALBERT
SentencePieceРаботает на уровне «сырых» символов без предварительной сегментацииmT5, XLNet, Claude
Пример BPE для слова «токенизация» Начало: т, о, к, е, н, и, з, а, ц, и, я → слияния частых пар → ток, ен, из, ация → или: токен, изация. Конкретный результат зависит от обучающего корпуса.

4Токенизация в LLM

Как LLM обрабатывают текст

Пользователь вводит текст → токенизатор разбивает его на токены → каждый токен сопоставляется с числовым идентификатором из словаря (vocabulary, обычно 32 000–128 000 единиц) → числа подаются на вход нейросети. Модель «видит» не буквы и не слова, а последовательность числовых кодов токенов.

Контекстное окно и лимит токенов

Каждая LLM имеет ограничение на длину входа + выхода, измеряемое в токенах (например, 8K, 128K, 200K). Это значит:

  • Длинные тексты могут не поместиться целиком.
  • Языки с богатой морфологией (русский, финский, турецкий) «тратят» больше токенов на то же содержание, чем английский.
  • Стоимость API обычно рассчитывается за количество токенов.
Русский vs. английский: расход токенов Одно и то же сообщение на русском языке может занимать в 1,5–2 раза больше токенов, чем на английском. Причина: токенизаторы обучены преимущественно на англоязычных корпусах, русские словоформы чаще разбиваются на подсловные единицы.

Практические следствия для лингвиста

  • Ошибки модели на стыках токенов (особенно при работе с морфологией) имеют системный характер.
  • Редкие слова, неологизмы, имена собственные разбиваются на мелкие токены — модель обрабатывает их менее уверенно.
  • Понимание токенизации помогает формулировать промпты, минимизирующие ошибки.

5LLM как образовательный NPC

Концепция

Термин NPC (Non-Player Character) означает: LLM получает чётко определённую роль и действует в её рамках, вместо того чтобы быть «универсальным ассистентом».

Бессубъектный актор (агенc) — ключевое понятие. LLM не обладает субъектностью (сознанием, намерениями, опытом), но может последовательно имитировать поведение определённого персонажа. Задавая роль, мы не создаём субъекта — мы конфигурируем модель поведения.

Типы образовательных NPC

Тип NPCФункцияКогда использовать
ТокенизаторДемонстрирует и объясняет процесс разбиения текста на токеныИзучение основ NLP, знакомство с механикой LLM
НаставникНаправляет обучение, задаёт вопросы, поддерживает рефлексиюСамостоятельное изучение новой темы
ЭкспертДаёт глубокие, детальные объяснения в конкретной областиУглублённое изучение, подготовка к экзамену
КритикАнализирует работу студента, даёт конструктивную обратную связьПроверка эссе, перевода, плана урока

Преимущества ролевого подхода

  • Фокусировка: модель не отвлекается на посторонние темы.
  • Последовательность: ответы выдержаны в едином стиле и уровне сложности.
  • Педагогическая стратегия: можно задать, как NPC реагирует на ошибки — сразу исправляет, задаёт наводящие вопросы или ждёт самокоррекции.
  • Воспроизводимость: удачный промпт можно переиспользовать и дорабатывать.

6Стратегии промптинга для создания NPC

Структура промпта

Эффективный промпт для создания образовательного NPC включает четыре компонента:

  1. Роль — кто этот персонаж (специальность, должность, функция).
  2. Характер — стиль коммуникации (строгий, дружелюбный, сократический).
  3. Экспертиза — предметная область и уровень глубины.
  4. Стратегия взаимодействия — как реагировать на вопросы и ошибки.

Примеры промптов

Ты — NPC-токенизатор по имени «ТокенМастер».

Роль: специалист по токенизации текста для NLP.
Характер: увлечённый, терпеливый, любишь наглядные примеры.
Экспертиза: BPE, WordPiece, SentencePiece, особенности токенизации разных языков.
Стратегия: когда пользователь даёт текст, ты пошагово показываешь, как он разбивается на токены, объясняешь логику разбиения, сравниваешь с лингвистическим морфемным разбором.

Начни с приветствия и предложи пользователю ввести слово или фразу для анализа.
Ты — NPC-наставник по имени «Навигатор».

Роль: педагог-фасилитатор, помогающий студенту-лингвисту самостоятельно разобраться в теме токенизации и работы LLM.
Характер: тёплый, поддерживающий, применяешь сократический метод — предпочитаешь задавать вопросы, а не давать готовые ответы.
Экспертиза: прикладная лингвистика, NLP, методика преподавания.
Стратегия: узнай, что студент уже знает. Предлагай задания нарастающей сложности. После каждого ответа студента давай краткую обратную связь и задавай следующий вопрос. Не давай готовых ответов сразу — веди к самостоятельному открытию.
Ты — NPC-эксперт по имени «Профессор Лексикон».

Роль: авторитетный исследователь в области компьютерной лингвистики.
Характер: академичный, точный, приводишь ссылки на ключевые работы (Sennrich et al. 2016 — BPE; Kudo 2018 — SentencePiece; и др.).
Экспертиза: глубокое знание алгоритмов токенизации, архитектуры Transformer, связь между лингвистической теорией и NLP.
Стратегия: отвечай развёрнуто и структурированно. Используй терминологию, но поясняй её. Приводи конкретные числовые примеры (размеры словарей, длины последовательностей). Сравнивай подходы объективно.
Ты — NPC-критик по имени «Рецензент».

Роль: строгий, но справедливый рецензент студенческих работ в области прикладной лингвистики.
Характер: прямолинейный, конкретный, всегда обосновывает свою оценку. Отмечаешь и сильные стороны, и недостатки.
Экспертиза: академическое письмо, лингвистическая терминология, стандарты научных работ.
Стратегия: когда студент присылает текст, ты оцениваешь его по критериям: (1) точность терминологии, (2) полнота раскрытия темы, (3) логичность аргументации, (4) качество примеров. Даёшь итоговую оценку и 2–3 конкретные рекомендации по улучшению.

Сам промптинг по заданию роли студенты также могут и должны обсудить с LLM — это называется метапромптинг. Пример:

Я студент-лингвист, изучаю использование LLM в образовании. Помоги мне составить промпт, который превратит тебя в образовательного NPC — наставника по фонетике английского языка для русскоязычных студентов.

Требования к NPC:
— сократический метод обучения;
— фокус на произношении проблемных звуков (/θ/, /ð/, /ŋ/);
— использование IPA-транскрипции;
— уровень студента: B1.

Предложи промпт и объясни, почему ты выбрал именно такие формулировки.

Такой подход позволяет студенту не только получить готовый промпт, но и понять принципы его конструирования.

Совет по итеративному улучшению Первый промпт редко бывает идеальным. После нескольких реплик оцените, насколько NPC следует заданной роли. Если поведение отклоняется, уточните промпт: добавьте ограничения, конкретизируйте стратегию, приведите примеры желаемых ответов.

7Ограничения и риски использования LLM как NPC-токенизатора

«Все модели неверны, но некоторые полезны» — Джордж Бокс Этот принцип статистика Джорджа Бокса в полной мере относится к LLM в роли токенизатора. Модель способна дать полезное приближение и объяснение, но её ответы не являются точной репликой реального алгоритма. Осознание границ модели — не менее важный учебный результат, чем освоение самой токенизации.

1. LLM не выполняет настоящую токенизацию

Когда вы просите LLM разбить текст на токены, она не запускает токенизатор внутри себя, а генерирует правдоподобный ответ на основе статистических паттернов. Это принципиальное различие:

Реальный токенизаторLLM в роли NPC-токенизатора
Детерминированный алгоритм: один вход — всегда один и тот же выходСтохастическая генерация: при разных запусках результат может различаться
Работает с конкретным словарём (например, cl100k_base для GPT-4)Не имеет доступа к своему словарю токенов; «рассуждает» о нём приблизительно
Гарантированная точностьМожет допускать ошибки, особенно на редких словах и языках, отличных от английского
Мгновенный результат для любой длины текстаОграничена контекстным окном; длинные тексты требуют разбиения

2. Галлюцинации и ложная уверенность

LLM может выдать токенизацию, которая выглядит убедительно, но фактически неверна. Это проявление общего свойства языковых моделей — галлюцинаций. Особенно высок риск в следующих случаях:

  • Редкие и длинные слова — модель может разбить их произвольно, не по границам реального BPE-словаря.
  • Нелатинские алфавиты — токенизация кириллицы, арабского, CJK-символов хуже представлена в обучающих данных модели.
  • Подсчёт количества токенов — LLM систематически ошибается при подсчёте, поскольку не умеет надёжно считать элементы последовательностей.
  • Специальные символы и пробелы — обработка пунктуации, юникодных символов и пробелов часто описывается моделью неточно.
Правило верификации Любой результат, полученный от LLM-токенизатора, следует проверять с помощью настоящего инструмента. Эталонные ресурсы: OpenAI Tokenizer (platform.openai.com/tokenizer) для моделей GPT, библиотека tiktoken (Python), sentencepiece для моделей на базе T5/LLaMA. Для токенизатора Claude — библиотека от Anthropic.

3. Отсутствие доступа к собственным параметрам

LLM не может «заглянуть» в свой собственный токенизатор. Она не знает:

  • точный состав своего словаря токенов;
  • какие именно ID присвоены каким подсловным единицам;
  • как конкретно был токенизирован входной промпт пользователя.

Всё, что она «знает» о токенизации, — это информация из обучающего корпуса: статьи, документация, учебники. Она воспроизводит знание о токенизации, а не саму токенизацию.

6. Итог

Использование LLM как образовательного NPC-токенизатора — это модель в боксовском смысле: она неверна в деталях, но полезна для формирования интуиции и концептуального понимания. Задача преподавателя и студента — осознавать границы этой модели и дополнять её точными инструментами там, где требуется достоверность.

Критическое мышление при работе с LLM — это не скептицизм ради скептицизма, а профессиональная компетенция: умение извлекать пользу из несовершенного инструмента, понимая природу и масштаб его несовершенств.

8LLM как NPC в других областях препроцессинга текста

Подход «LLM как образовательный NPC» применим не только к токенизации. Препроцессинг текста включает ряд смежных задач, в каждой из которых LLM может выступать наставником или инструментом — с различной степенью надёжности.

«Все модели неверны, но некоторые полезны» — Джордж Бокс Во всех областях препроцессинга прослеживается устойчивый паттерн: LLM лучше рассуждает о процессе, чем воспроизводит процесс. Для студентов-лингвистов это само по себе ценное наблюдение: оно иллюстрирует разницу между декларативным знанием (знать, что) и процедурным знанием (знать, как) — различие, хорошо знакомое в лингводидактике.

1. Очистка текста

Удаление HTML-тегов, спецсимволов, технического шума. LLM в роли NPC-«чистильщика» — одна из наиболее сильных ролей, поскольку очистка допускает вариативность результата (нет единственного детерминированного алгоритма).

ПодзадачаКачество выполненияКачество объяснения
Удаление HTML-теговОтлично — модель хорошо «знает» структуру HTMLОтлично
Удаление спецсимволовХорошо, но может «потерять» значимые символыОтлично
Фильтрация шумаХорошо — способна отличить контент от служебного мусораОтлично
Риск: тихая потеря информации Модель может молча удалить значимую информацию — математические символы, эмодзи с семантической нагрузкой, специальную разметку — или, наоборот, оставить шум, приняв его за контент. Всегда сверяйте результат с исходным текстом.

2. Нормализация

Приведение текста к единообразному виду: регистр, Unicode-формы (NFC/NFD), пунктуация. Здесь LLM полезна для объяснений, но ненадёжна для выполнения.

ПодзадачаОбъяснениеВыполнение
Приведение регистраОтлично объясняет правила и исключенияВыполняет надёжно
Unicode-нормализацияХорошо объясняет разницу NFC/NFDНенадёжна — не различает визуально идентичные символы
Унификация пунктуацииХорошо объясняет правилаМожет применять непоследовательно
Наглядный пример ограничения Unicode-нормализация требует побайтовой точности. LLM не «видит» разницу между «й» (U+0439, один символ) и «и» + «̆» (U+0438 + U+0306, два символа). Модель работает на уровне токенов, а не на уровне кодовых точек Unicode. Именно на этом примере можно показать студентам, почему LLM не заменяет детерминированные инструменты.

3. Лемматизация и стемминг

Две родственные, но принципиально разные задачи. Отношение LLM к ним — асимметрично.

3.1. Лемматизация

Приведение словоформы к начальной форме (лемме) с учётом контекста. Здесь LLM парадоксально близка к экспертному инструменту, поскольку лемматизация требует понимания контекста — именно того, в чём языковые модели сильны.

ПримерПравильная леммаLLM справится?Комментарий
«бежавших»бежатьДа, уверенноСтандартная форма
«стали» (металл)стальДа — учитывает контекстПреимущество перед pymorphy2, который не различает омонимы без контекста
«стали» (глагол)статьДа — учитывает контекст
«печь» (сущ. vs. глагол)печь / печьДа — по контекстуКонтекстная дизамбигуация
Редкие / устаревшие формыСпецифичноМожет ошибатьсяНестабильность на периферии

3.2. Стемминг

Механическое отсечение окончаний и суффиксов (алгоритмы Портера, Snowball). Здесь LLM избыточна: она будет «рассуждать» над задачей, которая решается простым алгоритмом за микросекунды.

КритерийАлгоритмический стеммерLLM
СкоростьМикросекундыСекунды
ДетерминированностьАбсолютнаяНет — результат может варьироваться
Педагогическая ценностьНужно объяснять отдельноМожет объяснить и показать пошагово
Практическое применениеОправданоНе оправдано — проще запустить стеммер
Вывод LLM-лемматизатор полезнее LLM-стеммера. Для стемминга LLM ценна как объясняющий NPC (наставник, показывающий логику алгоритма), но не как выполняющий инструмент.

4. Удаление стоп-слов

Фильтрация служебных слов (предлоги, союзы, частицы), не несущих самостоятельной семантики. Интересный случай, где LLM потенциально превосходит традиционные инструменты.

ПодходТрадиционный фильтрLLM
Список стоп-словФиксированный (NLTK: около 150 слов для русского)Может рассуждать контекстно
«Не» — стоп-слово?Всегда удаляется по спискуМожет оставить, если семантически значимо
Доменная адаптацияТребует ручной настройки спискаМожет адаптироваться через промпт
Пример контекстного преимущества В предложении «Не всё то золото, что блестит» слово «не» несёт ключевую семантику и не должно удаляться, хотя формально входит в любой стоп-лист. LLM способна это распознать. Традиционный фильтр — нет.

Риск: непоследовательность. При обработке корпуса из 1000 предложений LLM может в одних случаях удалить слово, в других — оставить его, без объяснимой логики. Для воспроизводимости результатов это неприемлемо.

6. Педагогические рекомендации

ПринципРеализация
Разделяй роли Используйте LLM как NPC-наставника для объяснения всех областей, но как NPC-инструмент — только для очистки и лемматизации, и всегда с верификацией
Сравнивай результаты Давайте студентам задания на сопоставление: результат LLM vs. результат алгоритмического инструмента (NLTK, spaCy, pymorphy2, BeautifulSoup)
Ищи границы Лучшее учебное упражнение — найти случай, где LLM ошибается. Это формирует критическое мышление и углубляет понимание предмета
Документируй Приучайте студентов фиксировать промпты, ответы модели и результаты верификации. Это основа воспроизводимости
Чередуй Каждое задание «с LLM» должно сопровождаться заданием «без LLM» — ручным анализом или использованием программного инструмента
Итог Во всех областях препроцессинга LLM максимально полезна как объясняющий NPC и ограниченно полезна как выполняющий NPC. Осознание этой асимметрии — ключевая компетенция для лингвиста, работающего в парадигме AI First. Студент, который понимает, когда и почему LLM надёжна, а когда нет, владеет инструментом на качественно ином уровне, чем тот, кто просто принимает любой ответ модели на веру.

9Интерактивная токенизация

Введите текст и посмотрите, как он разбивается на токены разными способами. Это упрощённая демонстрация — реальные токенизаторы LLM используют обученные BPE/SentencePiece-словари.

Задания для самостоятельной работы

  1. Введите в демонстрацию одно и то же предложение на русском и английском языке. Сравните количество токенов. Объясните разницу.
  2. Введите слово с приставкой и суффиксом (например, «перечитывание»). Сравните подсловную токенизацию с морфемным разбором.
  3. Откройте любую LLM (ChatGPT, Claude, GigaChat) и используйте один из промптов из раздела 6. Оцените, насколько модель следует заданной роли.
  4. Составьте собственный промпт для NPC, который помогает изучать грамматику изучаемого вами иностранного языка. Обсудите промпт с LLM (метапромптинг).
