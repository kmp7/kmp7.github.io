Теоретические основы токенизации

Токенизация — это процесс разбиения текста на минимальные смысловые единицы (токены), которые затем преобразуются в числовые представления для обработки нейронными сетями.

Токенизация является первым и критически важным этапом в работе больших языковых моделей (LLM). От выбора алгоритма токенизации зависит:

  • Размер словаря модели
  • Способность обрабатывать редкие и новые слова
  • Эффективность использования контекстного окна
  • Качество работы модели с разными языками
Токенизация по пробелам и пунктуации — простейший метод, разделяющий текст по пробельным символам и знакам препинания.

Алгоритм:

  1. Найти все пробельные символы и знаки препинания
  2. Разделить текст по найденным позициям
  3. Сохранить знаки препинания как отдельные токены

Преимущества:

  • Простота реализации
  • Интуитивная понятность результата

Недостатки:

  • Огромный размер словаря для морфологически богатых языков
  • Неспособность обрабатывать неизвестные слова (OOV проблема)
  • Неэффективность для агглютинативных языков
Посимвольная токенизация — разбиение текста на отдельные символы.

Алгоритм:

  1. Преобразовать строку в массив символов
  2. Каждый символ становится отдельным токеном

Преимущества:

  • Минимальный размер словаря
  • Отсутствие проблемы неизвестных слов
  • Универсальность для любого языка

Недостатки:

  • Очень длинные последовательности токенов
  • Потеря морфологической информации
  • Сложность обучения на длинных последовательностях
Byte Pair Encoding (BPE) — алгоритм сжатия данных, адаптированный для токенизации. Итеративно объединяет наиболее частые пары символов.

Алгоритм обучения:

  1. Начать с посимвольного разбиения корпуса
  2. Подсчитать частоту всех пар соседних токенов
  3. Объединить самую частую пару в новый токен
  4. Повторять шаги 2-3 до достижения желаемого размера словаря

Применение:

Используется в GPT-2, GPT-3, GPT-4, RoBERTa и многих других моделях.

WordPiece — алгоритм, разработанный Google, использующий вероятностную модель для выбора оптимального разбиения.

Особенности:

  • Использует префикс "##" для обозначения продолжения слова
  • Выбирает разбиение, максимизирующее правдоподобие корпуса
  • Более лингвистически мотивирован, чем BPE

Применение:

Используется в BERT, DistilBERT, ELECTRA и других моделях семейства BERT.

SentencePiece — библиотека от Google, реализующая токенизацию непосредственно на уровне Unicode-символов без предварительной сегментации по пробелам.

Ключевые особенности:

  • Языконезависимый подход — не требует предварительной токенизации
  • Использует специальный символ для пробела (обычно «▁»)
  • Поддерживает BPE и Unigram как внутренние алгоритмы
  • Обратимая токенизация — можно точно восстановить исходный текст

Алгоритм:

  1. Заменить пробелы на специальный символ «▁»
  2. Применить BPE или Unigram к полученной строке
  3. Сохранить информацию о границах слов в токенах

Применение:

Используется в T5, ALBERT, XLNet, mBART, многоязычных моделях.

Unigram Language Model — вероятностный алгоритм, который начинает с большого словаря и итеративно удаляет токены, минимально влияющие на правдоподобие корпуса.

Принцип работы:

  • Начинает с большого начального словаря (все подстроки до определенной длины)
  • Оценивает вклад каждого токена в общее правдоподобие
  • Удаляет токены с наименьшим влиянием
  • Повторяет до достижения целевого размера словаря

Особенности:

  • Может выдавать несколько вариантов сегментации с разными вероятностями
  • Более гибкий, чем BPE — не привязан к порядку слияний
  • Лучше обрабатывает морфологически сложные языки

Применение:

Часто используется внутри SentencePiece, применяется в ALBERT, T5, mBART.
