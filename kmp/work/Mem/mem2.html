Parametric memory LLM

долговременная память больших языковых моделей (веса = параметры нейросети)

Понятие «памяти» в контексте LLM

Метафора памяти в языковых моделях

В больших языковых моделях (LLM) термин «память» используется метафорически и охватывает несколько технологических уровней хранения и обработки информации. В отличие от человеческой памяти, «память» LLM — это совокупность различных механизмов, каждый из которых решает свою задачу.

Понимание этих уровней критически важно для преподавателей иностранных языков: оно помогает объяснить студентам, почему модель «знает» одно, «забывает» другое и иногда «выдумывает» третье.

Уровни «памяти» LLM

  • Parametric memory — (model weights) LLM: знания, закодированные в числовых параметрах нейросети в процессе обучения. Это основа всего, что модель «знает».
  • Контекстное окно — «оперативная память»: информация из текущего диалога или обрабатываемого текста в пределах лимита токенов (например, 8 000, 32 000 или 128 000 токенов в зависимости от модели).
  • KV-Cache (Key-Value Cache) — технический кэш промежуточных вычислений механизма внимания для ускорения генерации. Существует только во время одной сессии генерации.
  • Внешние механизмы (RAG, векторные БД) — доступ к актуальным данным через поиск. Информация не хранится внутри модели, но может использоваться в контексте запроса.
  • Сессионная и глобальная память сервиса — метаданные пользователя (настройки, история диалогов, предпочтения), сохраняемые платформой, а не самой моделью.
Важно: «Память» LLM — это каскад уровней с разными сроками жизни, правилами доступа и целями. Модель не «помнит» в человеческом смысле: она не имеет эпизодической памяти о прошлых диалогах (если это не реализовано на уровне сервиса) и не может самостоятельно обновлять свои знания.

Parametric memory (model weights) в структуре памяти LLM

Что такое parametric memory

Parametric memory — это основной уровень хранения знаний в больших языковых моделях (LLM), представленный весами модели (model weights). Эти веса представляют собой числовые параметры нейронной сети, оптимизированные в процессе обучения на огромных массивах текстовых данных.

В отличие от внешней (контекстной) памяти, parametric memory является статической: она формируется на этапе предобучения и не изменяется при инференсе (использовании модели).

Как кодируются знания в parametric memory

Знания кодируются неявно через настройку весов соединений между нейронами в многослойной архитектуре трансформера. Процесс кодирования включает следующие этапы:

  • оптимизацию весов в ходе обратного распространения ошибки (backpropagation);
  • минимизацию функции потерь (loss function) через градиентный спуск;
  • формирование распределённых представлений (distributed representations) в скрытых слоях;
  • создание взаимосвязей между входными токенами и выходными предсказаниями через многослойные нелинейные преобразования.
Пояснение: каждый вес в сети представляет собой числовой параметр (обычно типа float32 или float16), определяющий силу связи между нейронами. Совокупность всех весов формирует многомерное пространство параметров $\theta \in \mathbb{R}^n$, где $n$ — количество параметров модели (может достигать сотен миллиардов).

Что хранится в parametric memory

В parametric memory закодированы различные типы лингвистических и фактических знаний:

  • Лингвистические правила: грамматика, синтаксис, морфология, семантика.
  • Статистические закономерности: частотность слов и словосочетаний, коллокации.
  • Фактические знания: факты о мире, исторические события, научные данные (в объёме обучающих данных).
  • Контекстные зависимости: способность понимать и генерировать текст в зависимости от контекста.
  • Смысловые векторы: эмбеддинги слов и фраз, сохраняющие семантические отношения (например, векторное представление слова «король» будет ближе к «королеве», чем к «камню»).
  • Паттерны рассуждения: базовые логические и причинно‑следственные связи.
Компонент модели Тип хранимых знаний Механизм кодирования
Эмбеддинг‑слои Лексические и семантические представления Векторные представления токенов $E \in \mathbb{R}^{V \times d}$, где $V$ — размер словаря, $d$ — размерность эмбеддинга
Attention‑слои Контекстные зависимости между токенами Веса внимания $W_Q, W_K, W_V$ для вычисления self‑attention
Feed‑forward слои Нелинейные преобразования и паттерны Матрицы весов $W_1 \in \mathbb{R}^{d \times d_{ff}}, W_2 \in \mathbb{R}^{d_{ff} \times d}$
Нормализация (LayerNorm) Статистики активации Параметры сдвига $\beta$ и масштабирования $\gamma$
Примечание: объём parametric memory напрямую определяется количеством параметров модели. Например, GPT‑3 имеет $175$ миллиардов весов, каждый из которых вносит вклад в общее «знание» модели.
Важно: знания в parametric memory не хранятся в явной форме (как в базе данных), а распределены по миллионам/миллиардам весов. Извлечение конкретного факта требует активации соответствующей комбинации параметров через входной запрос.

Distributed representation

Принцип распределённого представления

В отличие от традиционных баз данных, где информация хранится в явном виде (например, «Париж — столица Франции»), в LLM знания распределены по миллионам или миллиардам весов модели. Это называется распределённым представлением (distributed representation).

Каждый вес сам по себе не несёт смысловой нагрузки — значение имеет только их совокупность и взаимосвязи между ними. Это похоже на работу человеческого мозга: воспоминание не хранится в одном нейроне, а активируется сетью нейронов.

Как происходит кодирование знаний

Процесс кодирования знаний в весах модели происходит во время обучения:

  • модель получает входные данные (текст) и пытается предсказать следующее слово;
  • вычисляется ошибка между предсказанием и реальным словом;
  • с помощью алгоритма обратного распространения ошибки (backpropagation) корректируются веса;
  • через миллионы итераций модель настраивает веса так, чтобы минимизировать ошибку на всём обучающем датасете.

В результате этой оптимизации в весах модели закодированы:

  • статистические закономерности языка;
  • семантические отношения между словами;
  • фактические знания из обучающих данных;
  • правила грамматики и синтаксиса.

Механизм извлечения знаний

Извлечение конкретного факта из parametric memory — это процесс активации определённой комбинации весов через входной запрос. Разберём его пошагово:

  1. Токенизация: входной текст разбивается на токены.
  2. Эмбеддинг: каждый токен преобразуется в векторное представление (эмбеддинг).
  3. Прохождение через слои: векторы проходят через слои трансформера, где каждый слой применяет свои веса к данным.
  4. Внимание (attention): механизмы внимания определяют, какие части входного текста наиболее релевантны для текущего предсказания.
  5. Нелинейные преобразования: feed‑forward слои выполняют сложные нелинейные преобразования данных.
  6. Нормализация: LayerNorm стабилизирует активации.
  7. Финальное предсказание: последний слой генерирует распределение вероятностей по словарю для следующего токена.
Пояснение: когда вы задаёте вопрос «Какая столица Франции?», модель не ищет ответ в базе данных. Вместо этого запрос активирует определённую комбинацию весов, которая была настроена во время обучения на примерах, где «Франция» и «Париж» встречались вместе. Эта активация приводит к тому, что слово «Париж» получает наибольшую вероятность в финальном распределении.
Этап Что происходит Роль весов
Вход Текст преобразуется в последовательность токенов Веса эмбеддинг‑слоя кодируют слова в векторы
Attention Определяются релевантные части контекста Веса матриц $W_Q, W_K, W_V$ настраивают внимание
FFN Выполняются нелинейные преобразования Веса $W_1, W_2$ кодируют паттерны рассуждений
Выход Генерируется распределение вероятностей Финальные веса определяют вероятности токенов

Примеры активации знаний

Рассмотрим, как разные запросы активируют разные комбинации весов:

  • Вопрос о столице: «Столица Франции?» → активируются веса, настроенные на географические факты.
  • Грамматический вопрос: «Как правильно: „едь“ или „поезжай“?» → активируются веса, кодирующие правила русского языка.
  • Логическая задача: «Если все кошки — животные, а Мурзик — кошка, то Мурзик…» → активируются веса, кодирующие простые логические правила.
  • Творческое задание: «Напиши стихотворение про робота» → активируются веса, отвечающие за поэтический стиль и лексику.
Примечание: один и тот же вес может участвовать в активации разных знаний. Например, вес, помогающий ответить на вопрос о столице Франции, может также участвовать в ответах о европейской географии, истории Франции или французской культуре. Это свойство называется полифункциональностью весов.
Важно: из‑за распределённого характера хранения знаний:
  • невозможно точно определить, где именно хранится конкретный факт;
  • удаление одного факта без влияния на другие практически невозможно;
  • точность извлечения зависит от формулировки запроса (разные формулировки активируют разные комбинации весов);
  • модель может «забывать» редкие факты, если они редко встречались в обучении.

Этапы обучения и их вклад в «долговременную память»

Общая схема обучения LLM

«Долговременная память» LLM формируется не за один этап, а в результате последовательного обучения, где каждый этап вносит свой специфический вклад. Можно провести аналогию с образованием человека: от общего образования к специализации и профессиональной этике.

Основные этапы обучения современных LLM:

  • Pretraining (предварительное обучение) — фундамент знаний
  • Supervised Fine-Tuning (SFT) — обучение формату взаимодействия
  • Reinforcement Learning from Human Feedback (RLHF) — настройка поведения
  • Дополнительные этапы — специализация и актуализация
Этап Аналогия из образования Главный вклад Объём данных
Pretraining Общее образование (школа + университет) «Что существует в мире» Триллионы токенов
SFT Профессиональная подготовка «Как отвечать на вопросы» Тысячи–миллионы примеров
RLHF Этика и деонтология профессии «Что можно и нельзя говорить» Сотни тысяч оценок
Domain Fine-Tuning Узкая специализация (ординатура, аспирантура) Глубокие знания в области Варьируется
Примечание: Не все модели проходят все этапы. Базовые модели (base models) проходят только Pretraining. Модели для диалога (chat models, instruct models) дополнительно проходят SFT и часто RLHF.

Pretraining (предварительное обучение)

Суть предварительного обучения

Pretraining — это первый и самый масштабный этап обучения LLM. Модель обучается на огромных массивах неструктурированных текстов с единственной задачей: предсказывать следующий токен (слово или его часть) на основе предыдущих.

Источники данных для pretraining включают:

  • Веб-страницы (Common Crawl и другие архивы интернета)
  • Книги (художественная и научная литература)
  • Научные статьи и журналы
  • Код из открытых репозиториев (GitHub)
  • Википедия и другие энциклопедии
  • Новостные архивы
  • Субтитры, диалоги, форумы
Пояснение: Задача «предсказать следующее слово» кажется простой, но для её качественного выполнения модель вынуждена неявно усваивать грамматику, семантику, логику, факты о мире и даже элементы здравого смысла.

Вклад в «память» модели

На этапе pretraining формируются:

  • Базовые языковые закономерности — грамматика, синтаксис, морфология, стилистические особенности разных жанров и регистров
  • Широкая семантика — значения слов и фраз в различных контекстах, полисемия, синонимия, антонимия
  • «Общие знания» о мире — факты, события, понятия, отношения между сущностями (без гарантии точности и актуальности)
  • Эмбеддинги (embeddings) — векторные представления слов и понятий, отражающие их семантические связи в многомерном пространстве
  • Многоязычные связи — для мультиязычных моделей: соответствия между языками, переводные эквиваленты
Характеристика «памяти» после Pretraining Описание
Охват тем Максимально широкий — от квантовой физики до кулинарных рецептов
Глубина знаний Поверхностная по большинству тем, глубже там, где больше данных
Точность Не гарантирована — модель воспроизводит паттерны из данных, включая ошибки
Актуальность Ограничена датой «заморозки» (knowledge cutoff) обучающих данных
Формат ответов Не сформирован — модель склонна «продолжать текст», а не «отвечать на вопросы»

Значение для лингвистов

Для специалистов в области языка особенно важны следующие аспекты pretraining:

  • Дистрибутивная гипотеза в действии: модель буквально реализует принцип «слово определяется своим окружением» (Дж. Фёрс, З. Харрис)
  • Статистическая природа «знания» языка: модель усваивает не правила, а вероятностные закономерности
  • Языковой дисбаланс: английский язык доминирует в обучающих данных, что влияет на качество работы с другими языками
  • Регистры и стили: модель усваивает различия между формальным и неформальным языком, жанровые особенности
Важно: После pretraining модель ещё не является «ассистентом». Она умеет продолжать тексты в стиле обучающих данных, но не умеет целенаправленно отвечать на вопросы, следовать инструкциям или вести диалог. Для этого нужны следующие этапы обучения.
Примечание: Явление «галлюцинаций» (генерации правдоподобного, но ложного контента) частично коренится именно в pretraining: модель учится генерировать убедительный текст, а не проверять факты. Она воспроизводит статистические паттерны, даже если они основаны на ошибочных данных.

Supervised Fine-Tuning (SFT) — дообучение с учителем

Суть дообучения с учителем

Supervised Fine-Tuning (SFT) — это этап дообучения предварительно обученной модели на меньшем, но качественно размеченном наборе данных. Данные представляют собой пары «инструкция/запрос → эталонный ответ».

Примеры обучающих пар для SFT:

  • «Напиши краткое эссе о фотосинтезе» → [качественное эссе о фотосинтезе]
  • «Переведи на французский: The weather is beautiful today» → «Le temps est magnifique aujourd'hui»
  • «Исправь грамматические ошибки в тексте: ...» → [исправленный текст]
  • «Объясни разницу между Present Perfect и Past Simple для студента уровня B1» → [методически грамотное объяснение]
Пояснение: SFT можно сравнить с этапом, когда выпускник университета проходит практику под руководством опытного специалиста: он уже обладает знаниями, но учится применять их в профессиональном контексте по образцам.

Вклад в «память» модели

На этапе SFT модель усваивает:

  • Формат взаимодействия — модель учится распознавать структуру «вопрос-ответ», «инструкция-выполнение», «диалог»
  • Структура и стиль ответов — полнота, ясность, вежливость, уместный уровень детализации
  • Специализация на задачах — перевод, суммаризация, написание кода, ответы на вопросы, генерация текстов определённых жанров
  • Шаблоны и формулировки — типичные способы начинать ответ, структурировать объяснение, завершать диалог
  • Следование инструкциям — способность учитывать ограничения и требования из запроса («кратко», «для детей», «в формате списка»)
До SFT (только Pretraining) После SFT
Модель продолжает текст в стиле обучающих данных Модель отвечает на вопросы и выполняет инструкции
Может «застрять» в генерации или уйти в сторону Генерирует завершённые, структурированные ответы
Стиль непредсказуем (может быть формальным, разговорным, техническим) Стиль соответствует ожиданиям пользователя
Не понимает роль «ассистента» Ведёт себя как помощник, ориентированный на пользователя

Примеры SFT для лингвистических задач

В контексте преподавания языков SFT особенно важен для следующих навыков модели:

  • Адаптация объяснений под уровень: «Объясни герундий для уровня A2» vs «Объясни герундий для магистранта-филолога»
  • Генерация упражнений: создание заданий определённого типа и сложности
  • Обратная связь: корректировка ошибок учащихся с объяснениями
  • Перевод с учётом контекста: не дословный, а адекватный перевод
  • Работа с метаязыком: использование лингвистической терминологии уместным образом
Важно: SFT не добавляет принципиально новых знаний о мире — он учит модель как использовать знания из Pretraining в формате, полезном для пользователя. Если модель не встречала информацию о каком-то явлении на этапе Pretraining, SFT не поможет ей «узнать» об этом явлении.
Примечание: Качество SFT критически зависит от качества обучающих примеров. Если эталонные ответы содержат ошибки, предвзятость или неудачные формулировки, модель усвоит и их. Поэтому создание данных для SFT — трудоёмкий процесс с участием экспертов.

RLHF — обучение с подкреплением на основе обратной связи от людей

Суть метода RLHF

Reinforcement Learning from Human Feedback (RLHF) — это этап тонкой настройки модели с использованием обучения с подкреплением. Цель — научить модель генерировать ответы, которые люди оценивают как более качественные, безопасные и полезные.

Процесс RLHF включает несколько шагов:

  • Шаг 1: Модель генерирует несколько вариантов ответа на один запрос
  • Шаг 2: Люди-асессоры ранжируют варианты от лучшего к худшему
  • Шаг 3: На основе этих рейтингов обучается модель вознаграждения (Reward Model) — она учится предсказывать, какой ответ понравится людям
  • Шаг 4: LLM дообучается генерировать ответы, максимизирующие оценку модели вознаграждения
Пояснение: RLHF можно сравнить с наставничеством: модель получает обратную связь не в форме «правильно/неправильно», а в форме «этот ответ лучше, чем тот» — и постепенно учится соответствовать ожиданиям.

Вклад в «память» и поведение модели

RLHF формирует поведенческий слой «памяти»:

  • Безопасность: отказ от генерации вредного контента (инструкции по созданию оружия, пропаганда насилия и т.д.)
  • Полезность: стремление дать информативный, релевантный ответ
  • Честность: признание неуверенности, отказ от выдумывания («Я не уверен», «У меня нет информации об этом»)
  • Этичность: избегание дискриминации, оскорблений, предвзятости
  • Естественность: живой, неформальный стиль там, где это уместно
Аспект До RLHF После RLHF
Вредный контент Может генерировать, если есть в данных Отказывается или переформулирует
Неуверенность Генерирует уверенный текст даже при незнании Чаще признаёт ограничения
Спорные темы Может занять крайнюю позицию Стремится к сбалансированности
Тон общения Может быть резким или неуместным Вежливый, уважительный

Особенности RLHF

Ключевые характеристики этого этапа:

  • Не добавляет новых фактов: модель не узнаёт новую информацию о мире, но меняет способ подачи уже известного
  • Субъективность оценок: «хороший ответ» — понятие, зависящее от культуры, контекста, личных предпочтений асессоров
  • Риск «переобучения на угождение»: модель может научиться давать «приятные», но не обязательно точные ответы
  • Сложность формализации: RLHF решает задачи, которые трудно описать правилами (что такое «естественный» ответ?)
Важно: RLHF — это не цензура в традиционном понимании. Модель не получает список «запрещённых тем». Вместо этого она обучается на примерах человеческих предпочтений и вырабатывает обобщённые «поведенческие правила». Это объясняет, почему ограничения иногда срабатывают неожиданно или непоследовательно.
Примечание: Альтернативы и дополнения к RLHF: DPO (Direct Preference Optimization) — более простой метод, не требующий отдельной модели вознаграждения; RLAIF (RL from AI Feedback) — использование другой модели вместо людей для генерации оценок; Constitutional AI — обучение модели следовать явно сформулированным принципам.

Дополнительные этапы обучения

Domain-Specific Fine-Tuning (специализированное дообучение)

Дополнительное дообучение модели на данных из узкой предметной области для углубления экспертизы.

Примеры областей:

  • Медицина: обучение на клинических записях, медицинской литературе (модели BioGPT, Med-PaLM)
  • Юриспруденция: законодательные акты, судебные решения, договоры
  • Финансы: отчётность, аналитика, регуляторные документы
  • Лингвистика и преподавание языков: методические материалы, учебники, корпуса ошибок учащихся

Вклад в «память»:

  • Глубокое усвоение специализированной терминологии
  • Понимание специфических закономерностей и практик области
  • Более точные и профессиональные ответы в рамках домена

LoRA (Low-Rank Adaptation) и методы эффективной адаптации

LoRA — метод, позволяющий адаптировать модель к новым задачам без полного переобучения всех параметров.

Принцип работы:

  • Основные веса модели «замораживаются» (не изменяются)
  • К модели добавляются небольшие обучаемые матрицы-«адаптеры»
  • Обучаются только эти адаптеры (обычно менее 1% от общего числа параметров)

Преимущества:

  • Значительно меньше вычислительных ресурсов
  • Можно хранить множество «адаптеров» для разных задач
  • Легко переключаться между специализациями

Вклад в «память»: быстрое приобретение новых навыков или знаний без риска «забыть» базовые способности.

Пояснение: LoRA можно сравнить с изучением нового предмета специалистом: он не переучивается «с нуля», а добавляет новые знания к существующей базе.

Continual Learning / Online Learning (непрерывное обучение)

Периодическое дообучение модели на новых данных для актуализации знаний.

Цели непрерывного обучения:

  • Включение информации о событиях после даты «заморозки» (knowledge cutoff)
  • Исправление выявленных ошибок и «галлюцинаций»
  • Адаптация к изменениям в языке (новые слова, значения, нормы)
  • Учёт обратной связи от пользователей

Проблемы непрерывного обучения:

  • Катастрофическое забывание: новое обучение может «стереть» ранее усвоенные знания
  • Ресурсоёмкость: полное переобучение больших моделей крайне дорого
  • Контроль качества: сложно гарантировать, что обновление не внесёт новых проблем
Метод Когда применяется Вклад в «память» Ресурсоёмкость
Domain Fine-Tuning Нужна экспертиза в узкой области Глубокие специализированные знания Средняя — высокая
LoRA Быстрая адаптация, ограниченные ресурсы Специфические навыки без потери базовых Низкая
Continual Learning Устаревание знаний, накопление ошибок Актуализация, исправления Высокая
Примечание: Для преподавателей языков особенно перспективен Domain Fine-Tuning на корпусах ошибок учащихся и методических материалах. Такая модель может лучше понимать типичные ошибки русскоязычных (или других) учащихся и давать более целенаправленную обратную связь.

Итоги: формирование «долговременной памяти» LLM

Ключевые выводы

«Долговременная память» LLM — это результат всей цепочки обучения, а не только Pretraining.

  • Pretraining даёт фундамент — широкие, но поверхностные знания о языке и мире
  • SFT формирует «профессиональные навыки» — умение отвечать, объяснять, выполнять задачи
  • RLHF настраивает «личность» и поведение — безопасность, вежливость, этику
  • Дополнительные этапы обеспечивают специализацию и актуализацию
Этап Метафора Что даёт Тип знаний
Pretraining Общее образование «Что существует в мире» Факты, язык, закономерности
SFT Профессиональная практика «Как сказать» Формат, структура, стиль
RLHF Этика профессии «Что можно и нельзя говорить» Нормы, ограничения, предпочтения
Domain FT Узкая специализация Экспертиза в области Терминология, практики домена

Судьба весов модели

Важно понимать жизненный цикл параметров модели:

  • Во время обучения (любого этапа) — веса активно изменяются, модель «учится»
  • После развёртывания для пользователей — веса «замораживаются», модель работает в режиме инференса (применения)
  • При взаимодействии с пользователями — модель не обучается на диалогах, веса не меняются
Пояснение: Когда вы общаетесь с ChatGPT, Claude или другой LLM, модель не «запоминает» ваш разговор в своих весах. Всё «запоминание» происходит только через контекстное окно (в рамках одной сессии) или через внешние механизмы памяти на уровне сервиса.

Актуализация и специализация

  • Актуализация знаний возможна только через дообучение (Continual Learning) или внешние механизмы (RAG). Модель не «узнаёт» новое из диалогов с пользователями.
  • Специализация достигается на этапах SFT и Domain-Specific Fine-Tuning, что позволяет одной базовой модели (например, Llama-3, Mistral) иметь множество версий под разные задачи (чат, код, медицина, юриспруденция).
  • Knowledge cutoff (дата заморозки знаний) — критически важный параметр: модель не располагает информацией о событиях после этой даты (если не использует RAG или не была переобучена).
Важно для преподавателей: Понимание многослойной природы «памяти» LLM позволяет: (1) объяснять студентам, почему модель «знает» одно и «не знает» другое; (2) грамотно формулировать промпты, опираясь на сильные стороны модели; (3) критически оценивать ответы модели, понимая источники возможных ошибок; (4) осознанно выбирать между разными моделями и сервисами для конкретных задач.
Примечание: Ландшафт LLM быстро меняется. Появляются новые методы обучения (DPO, ORPO), новые архитектуры (Mixture of Experts), новые подходы к памяти (MemGPT, Retrieval-Augmented Generation). Базовые принципы, изложенные в этом материале, остаются актуальными, но конкретные реализации могут отличаться в разных моделях.
